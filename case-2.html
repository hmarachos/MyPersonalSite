Дообучение LLM-модели с заданным стилем

Как мы создали русскоязычную модель с собственным стилем общения и запустили её локально без передачи данных во внешние сервисы

В чём была проблема

Стандартные LLM-модели не подходили для специфических задач:

  • Готовые модели отвечают в общем стиле, не соответствующем вашему бренду
  • Передача данных в облачные сервисы (ChatGPT, Claude) создаёт риски конфиденциальности
  • Зависимость от внешних API и их стоимость
  • Невозможность контролировать поведение модели и её ответы
  • Нерусскоязычные модели выдают бессмысленный текст при работе с кириллицей

Итог: нужна собственная модель, которая работает локально, говорит на русском и отвечает в нужном стиле.

Что мы сделали

Реализовали полный цикл дообучения открытой LLM-модели:

  • Выбрали русскоязычную базовую модель (rugpt3small от Sberbank AI)
  • Собрали датасет в формате «вопрос–ответ» с нужным стилем общения
  • Применили LoRA (Low-Rank Adaptation) для эффективного дообучения
  • Запустили модель локально на обычном компьютере
  • Создали интерфейс для взаимодействия с дообученной моделью

Результат: полностью контролируемая модель, работающая локально, без передачи данных во внешние сервисы.

Как это работает на практике

  1. Подготовка датасета: собираем пары «вопрос–ответ» в JSON-формате с нужным стилем общения
  2. Токенизация: преобразуем текст в токены, которые понимает модель
  3. Настройка LoRA: создаём адаптеры (дополнительные веса) поверх базовой модели
  4. Дообучение: модель учится на примерах из датасета, запоминая стиль и структуру ответов
  5. Сохранение весов: дообученные веса сохраняются отдельно (всего ~46 МБ)
  6. Загрузка модели: при запуске берётся базовая модель и подгружаются LoRA-веса
  7. Генерация ответа: модель генерирует токены, которые преобразуются обратно в текст

Технические детали

Почему rugpt3small?

  • Русскоязычная модель, обученная на кириллице
  • Базируется на архитектуре GPT-2 (проверенная и стабильная)
  • Минимальные требования к ресурсам (работает на любом компьютере)
  • Быстрое дообучение и генерация ответов

Почему LoRA?

  • Эффективное дообучение без полной переподготовки модели
  • Дополнительные веса занимают минимум места
  • Быстрое обучение даже на слабом оборудовании
  • Можно создавать несколько адаптеров для разных стилей

Что это даёт вашему бизнесу

  • Полная конфиденциальность: данные остаются на вашем сервере, не передаются никуда
  • Контроль стиля: модель отвечает ровно так, как вы её обучили
  • Экономия: нет платежей за API-запросы, модель работает локально
  • Скорость: нет задержек на передачу данных в облако
  • Масштабируемость: можно создавать разные версии модели для разных задач
  • Портфолио: готовый кейс для демонстрации навыков в ML и NLP
